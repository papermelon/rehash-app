# Contributing to Rehash

Thank you for your interest in contributing to Rehash! This document provides guidelines and instructions for contributing.

## Development Setup

1. Fork the repository
2. Clone your fork: `git clone https://github.com/your-username/rehash-app.git`
3. Install dependencies: `npm install`
4. Copy `.env.example` to `.env.local` and fill in your values
5. Run the development server: `npm run dev`

## Code Style

- Use TypeScript for all new files
- Follow the existing code structure and naming conventions
- Use Prettier for formatting (run `npm run format` if configured)
- Write meaningful commit messages

## Making Changes

1. Create a new branch: `git checkout -b feature/your-feature-name`
2. Make your changes
3. Test thoroughly
4. Commit with clear messages: `git commit -m "Add feature: description"`
5. Push to your fork: `git push origin feature/your-feature-name`
6. Open a Pull Request

## Pull Request Guidelines

- Describe what your PR does and why
- Reference any related issues
- Include screenshots for UI changes
- Ensure all tests pass
- Keep PRs focused on a single feature or fix

## Areas for Contribution

- **Features**: New export formats, additional AI models, batch processing
- **UI/UX**: Improved animations, accessibility enhancements, mobile optimization
- **Documentation**: Tutorials, API docs, troubleshooting guides
- **Testing**: Unit tests, integration tests, E2E tests
- **Performance**: Optimization, caching strategies, lazy loading

## Questions?

Open an issue for discussion before starting major changes.
